# 互联网医院基础管理模块交付总结

## 交付范围

本次总控开发围绕 `hlw` 互联网医院 MVP，完成数据库、后端接口、前端页面、菜单入口、接口脚本和 README 文档的基础闭环。当前分支为 `feat/internet-hospital-mvp`。

## 已完成模块

- 认证与登录资料：登录、退出、登录用户资料回查。
- 系统基础管理：租户管理、用户管理、角色管理、菜单管理、字典管理、参数配置、岗位管理、权限管理、用户角色授权、角色菜单授权。
- 医生与科室管理：医生名录、科室管理、医生状态、医生科室绑定、排班管理、挂号费规则。
- 患者管理：患者资料、健康档案。
- 预约管理：预约单、号源池、支付、签到、抢单、放号配置。
- 问诊管理：问诊单、接单、延长、完成、问诊消息、WebSocket 消息落库。
- 处方管理：处方草稿、提交、审核通过、驳回、处方明细。
- 药品库存：药品资料、库存记录、配送发货。
- 订单中心：订单创建、订单列表、模拟支付、支付事件。
- 本地消息：`local_message` 数据库存储兜底，保留内存实现用于轻量上下文和单元测试。

## 数据库使用情况

当前统一 SQL `resources/sql/init.sql` 覆盖 30 张表：

```text
sys_tenant, sys_user, sys_role, sys_menu, sys_dict, sys_config, sys_post, sys_user_post,
sys_permission, sys_user_role, sys_role_menu, doc_department, doc_doctor,
doc_doctor_department, doc_schedule, pat_patient, pat_health_record,
apt_appointment, apt_number_source, apt_number_source_release_config,
con_consult, con_message, pre_prescription, pre_prescription_item,
drug_info, drug_stock, drug_delivery, ord_order, local_message, con_consult_image
```

`con_consult_image` 用于后续图文问诊图片附件扩展；当前前后端消息接口没有图片上传地址和图片排序入参，因此暂不启用，避免产生无入口的伪业务数据。

静态审计结果：

- `resources/sql/init.sql`：30 张表，表注释和字段注释缺失数均为 0。
