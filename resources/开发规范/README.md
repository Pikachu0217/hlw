# 项目开发规范

本目录用于沉淀互联网医院项目的长期开发规范，供后续后端、前端、数据库、脚本和运维相关工作统一参考。

## 目录说明

- `01-Lombok使用规范.md`：Lombok 注解使用约定，重点说明 `@Data`、`@Slf4j` 等常用注解的适用范围与限制。
- `02-互联网医院项目开发规范.md`：基于当前仓库扫描整理的项目级开发规范，覆盖后端、前端、数据库、脚本、文档和交付检查清单。
- `03-项目通用规范.md`：从当前项目规则和实现中提取的通用规范，覆盖协作红线、后端分层、接口、数据库、认证租户、前端、脚本、Git 和交付检查。

## 使用原则

1. 新增规范文档时，文件名统一采用 `序号-中文名称.md` 格式。
2. 规范内容统一使用中文（简体）编写，示例代码中的标识符保持英文。
3. 规范如涉及项目已有约束，应与仓库根目录 `AGENTS.md` 保持一致。
4. 规范更新应与对应代码改动保持同步，避免规范与实现脱节。
5. 通用开发规则优先阅读 `03-项目通用规范.md`，历史项目级细则可继续参考 `02-互联网医院项目开发规范.md`，专项注解约束继续阅读 `01-Lombok使用规范.md`。
